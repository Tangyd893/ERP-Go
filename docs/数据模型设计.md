# 数据模型设计

## 1. 设计目标

本文档定义 ERP-Go 的数据模型设计原则、数据库拆分方式、核心表设计、库存与财务流水规则、索引、审计和数据一致性策略。实现时应以服务独立数据所有权为前提，禁止跨服务直接访问数据库表。

## 2. 数据库拆分原则

推荐每个服务使用独立数据库或独立 Schema。

| 服务 | 数据库或 Schema 建议 |
| --- | --- |
| 认证权限服务 | `erp_iam` |
| 租户组织服务 | `erp_tenant` |
| 商品服务 | `erp_product` |
| 渠道服务 | `erp_channel` |
| 订单服务 | `erp_order` |
| 库存服务 | `erp_inventory` |
| WMS 仓储服务 | `erp_warehouse` |
| TMS 物流服务 | `erp_transport` |
| 采购服务 | `erp_purchase` |
| 财务服务 | `erp_finance` |
| 报表服务 | `erp_report` 或分析库 |
| 文件服务 | `erp_file` |
| 通知服务 | `erp_notification` |

## 3. 通用字段

所有业务主表建议包含：

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `tenant_id` | 租户 ID |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |
| `created_by` | 创建人 |
| `updated_by` | 更新人 |
| `deleted_at` | 软删除时间 |
| `version` | 乐观锁版本号 |

关键业务表建议额外包含：

| 字段 | 说明 |
| --- | --- |
| `business_no` | 业务单号 |
| `status` | 业务状态 |
| `source_type` | 来源类型 |
| `external_id` | 外部系统 ID |
| `remark` | 备注 |

- 禁止使用浮点数存储金额。

## 13. 审计与日志数据

审计日志必须记录：

- 操作人。
- 租户。
- 操作对象。
- 操作类型。
- 操作前摘要。
- 操作后摘要。
- IP、设备、用户代理。
- 请求 ID 和追踪 ID。

必须审计的动作：

- 登录和登出。
- 权限变更。
- 导出数据。
- 库存调整。
- 财务冲销和调整。
- 店铺授权变更。
- 订单强制取消。

## 14. 索引设计原则

- 所有业务查询必须包含 `tenant_id`。
- 高频列表查询建立组合索引，例如租户、状态、时间。
- 外部幂等字段建立唯一索引。
- 状态历史、调用日志、轨迹事件等大表按时间分区或冷热分离。
- 报表查询不要直接压业务库，进入分析库或预聚合表。

## 15. 数据一致性规则

- 库存余额和库存流水定期对账。
- 财务流水和业务单据定期对账。
- 平台订单和系统订单定期对账。
- 物流轨迹和订单发货状态定期对账。
- 消息队列死信必须有人工补偿入口。
