GitHub 地址：[https://github.com/Tangyd893/ERP-Go.git](https://github.com/Tangyd893/ERP-Go.git)

## 推荐阅读顺序

1. 先阅读 [项目架构设计.md](./项目架构设计.md)，确认系统目标、边界、目录和技术路线。
2. 再阅读 [领域模型设计.md](./领域模型设计.md)，理解核心业务对象如何拆分。
3. 然后阅读 [微服务设计说明.md](./微服务设计说明.md)，明确各服务如何落地。
4. 接着阅读 [接口与事件设计.md](./接口与事件设计.md)，确定服务之间如何协作。
5. 继续阅读 [数据模型设计.md](./数据模型设计.md)，指导数据库迁移和数据访问层设计。
6. 最后阅读 [实施路线与工程规范.md](./实施路线与工程规范.md)，按阶段实现和验证。

## 设计约束

- 本文档集只完成分析和设计，不包含实际业务代码实现。
- 服务从第一阶段即按微服务建设，不采用单体过渡。
- 每个服务拥有独立业务边界和数据所有权，其他服务不得直接访问其数据库。
- 核心业务必须围绕对象、接口、聚合、状态机和领域事件建模。
- 库存和财务是系统一致性的核心，必须通过流水、幂等、事务和对账机制保证可追溯。
- 任何平台、物流商、海外仓、支付或税务系统都必须通过适配器隔离。
