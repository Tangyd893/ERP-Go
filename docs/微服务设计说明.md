# 微服务设计说明

## 1. 设计目标

本文档细化 ERP-Go 各微服务的职责边界、对象设计、应用服务、接口、依赖和实现注意事项。实现时应先建立服务骨架，再逐步填充领域对象、应用用例、数据库迁移和接口。

## 2. 服务通用分层

每个业务服务采用相同分层，保证团队协作和测试方式一致。

```text
服务/
  接口层/
  应用层/
  领域层/
  基础设施层/
  配置/
  数据库迁移/
  测试/
```

| 层 | 职责 |
| --- | --- |
| 接口层 | HTTP、gRPC、事件消费、参数校验、响应转换 |
| 应用层 | 用例编排、事务边界、权限校验、跨服务调用、事件发布 |
| 领域层 | 聚合、实体、值对象、领域服务、状态机、领域事件 |
| 基础设施层 | 数据库、缓存、消息队列、RPC、外部 API、对象存储 |
| 配置 | 服务端口、数据库、队列、日志、限流、外部系统配置 |
| 数据库迁移 | 服务私有表结构和初始化数据 |
| 测试 | 单元测试、集成测试、契约测试入口 |

## 3. API 网关

### 3.1 职责

- 统一外部入口。
- 路由到各业务服务。
- 校验 Token、租户、请求签名。
- 执行限流、熔断、灰度和黑白名单。
- 注入 `trace_id`、`request_id`、`tenant_id`、`user_id`。
- 统一错误响应和访问日志。

### 3.2 不应承担的职责

- 不保存业务数据。
- 不实现订单、库存、仓储等业务规则。
- 不绕过服务直接访问数据库。

### 3.3 关键对象

| 对象 | 职责 |
| --- | --- |
| 路由规则 | 定义路径、方法、目标服务和版本 |
| 鉴权策略 | 定义公开接口、登录接口、租户接口和权限接口 |
| 限流策略 | 按用户、租户、IP、接口、服务做限流 |
| 网关上下文 | 保存请求 ID、追踪 ID、租户、用户、语言和时区 |

## 4. 认证权限服务

### 4.1 职责

- 用户登录、登出、Token 刷新。
- 用户、角色、权限、菜单、按钮权限。
- 数据权限、仓库权限、店铺权限。
- 操作审计和登录日志。

### 4.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 登录应用服务 | 校验账号密码、生成 Token、记录登录日志 |
| 权限应用服务 | 查询用户功能权限和数据权限 |
| 用户管理应用服务 | 用户创建、禁用、重置密码、绑定角色 |
| 角色管理应用服务 | 角色创建、授权、权限变更 |
| 审计应用服务 | 写入和查询操作审计 |

### 4.3 对外接口

| 接口 | 类型 | 说明 |
| --- | --- | --- |
| 登录 | HTTP | 用户登录并获取 Token |
| 刷新 Token | HTTP | 使用刷新 Token 获取新访问 Token |
| 查询当前用户 | HTTP | 获取用户信息、角色、权限和数据范围 |
| 校验权限 | gRPC | 供网关或业务服务校验权限 |
| 写审计日志 | 事件或 gRPC | 记录关键操作 |

## 5. 租户组织服务

### 5.1 职责

- 租户、组织、部门、岗位、业务组。
- 租户套餐、功能开关、配额。
- 数据范围配置。

### 5.2 关键规则

- 所有业务请求必须携带租户上下文。
- 服务内部必须校验当前租户是否启用。
- 租户停用后，禁止新业务写入，但保留历史查询和导出能力。

### 5.3 对外接口

| 接口 | 类型 | 说明 |
| --- | --- | --- |
| 查询租户 | gRPC/HTTP | 获取租户状态、套餐、功能开关 |
| 查询组织树 | HTTP | 管理后台展示组织架构 |
| 查询数据范围 | gRPC | 供业务服务过滤店铺、仓库、供应商数据 |

## 6. 商品服务

### 6.1 职责

- 商品、SKU、变体、组合商品。
- 平台 SKU 映射。
- 包装规格、重量尺寸。
- 报关申报信息。

### 6.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| SKU 管理应用服务 | 创建、编辑、启用、停用 SKU |
| 商品映射应用服务 | 维护平台 SKU、店铺 SKU、ASIN、FNSKU 映射 |
| 组合商品应用服务 | 维护套装和子 SKU 关系 |
| 申报信息应用服务 | 维护 HS Code、申报名称、申报价值 |

### 6.3 依赖关系

- 被订单服务查询 SKU 信息。
- 被库存服务引用 SKU。
- 被仓储服务查询条码和包装信息。
- 被物流服务查询尺寸、重量和申报属性。

商品服务不依赖订单、库存、仓储服务，保持主数据独立。

## 7. 渠道服务

### 7.1 职责

- 店铺授权和授权刷新。
- 平台订单、商品、库存、退货、结算同步。
- 发货和运单号回传。
- 平台 API 限流、重试、死信和审计。

### 7.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 店铺授权应用服务 | 创建授权、刷新 Token、授权失效告警 |
| 订单同步应用服务 | 拉取平台订单并发布订单导入命令 |
| 库存推送应用服务 | 接收库存变化并向平台推送可售库存 |
| 发货回传应用服务 | 接收发货事件并回传平台 |
| 结算报告应用服务 | 创建、轮询、下载和解析平台结算报告 |

### 7.3 外部适配器

| 适配器 | 说明 |
| --- | --- |
| Amazon 适配器 | 对接 Amazon SP-API |
| Shopify 适配器 | 对接 Shopify API |
| eBay 适配器 | 对接 eBay API |
| Walmart 适配器 | 对接 Walmart API |

### 7.4 实现注意事项

- 外部 API 必须设置超时、重试、限流和熔断。
- 同步任务必须可暂停、可重试、可人工补偿。
- 平台返回错误码要转换为统一错误模型。
- 平台原始数据建议保存归档文件或摘要，便于排查。

## 8. 订单服务

### 8.1 职责

- 销售订单导入、查询、审核。
- 拆单、合单、取消、拦截。
- 异常订单处理。
- 售后、退款、补发、换货。

### 8.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 订单导入应用服务 | 按店铺和平台订单号幂等创建订单 |
| 订单审核应用服务 | 校验地址、库存、物流、利润和风控 |
| 拆单应用服务 | 按仓库、SKU、库存和物流限制拆分订单 |
| 订单取消应用服务 | 校验状态并触发库存释放或仓库拦截 |
| 售后应用服务 | 管理退款、退货、补发和换货 |

### 8.3 主要接口

| 接口 | 类型 | 说明 |
| --- | --- | --- |
| 创建订单 | gRPC/事件 | 渠道服务导入平台订单 |
| 审核订单 | HTTP | 管理后台或自动任务触发审核 |
| 查询订单 | HTTP | 多条件分页查询 |
| 取消订单 | HTTP | 用户或平台触发取消 |
| 接收库存锁定结果 | 事件 | 更新订单库存状态 |
| 接收出库结果 | 事件 | 更新订单发货状态 |

### 8.4 依赖

- 查询商品服务获取 SKU 映射。
- 调用库存服务申请锁定或释放。
- 接收 WMS 出库事件。
- 发布发货回传命令给渠道服务。

## 9. 库存服务

### 9.1 职责

- 库存余额。
- 库存锁定、释放、扣减。
- 调拨、盘点、调整。
- 库存流水和批次。

### 9.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 库存查询应用服务 | 查询可售、锁定、冻结、在途和可承诺库存 |
| 库存锁定应用服务 | 为订单锁定库存并写流水 |
| 库存释放应用服务 | 订单取消、出库失败、售后拦截时释放库存 |
| 库存扣减应用服务 | 仓库出库完成后扣减已锁定库存 |
| 库存调整应用服务 | 盘点、报损、冻结、解冻 |
| 调拨应用服务 | 管理跨仓和库内库存移动 |

### 9.3 并发控制

- SKU + 仓库 + 库位 + 批次 + 库存状态为库存余额锁粒度。
- 锁定、释放、扣减在同一个本地事务内完成。
- 库存操作必须携带业务幂等键。
- 每次变化必须写库存流水，余额更新失败则事务回滚。

## 10. WMS 仓储服务

### 10.1 职责

- 仓库、库区、库位。
- 入库、收货、质检、上架。
- 出库、波次、拣货、复核、打包、称重、交接。
- PDA 扫码作业。

### 10.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 入库应用服务 | 创建入库单、收货、质检、上架 |
| 出库应用服务 | 创建出库单、分配波次、完成出库 |
| 拣货应用服务 | 生成拣货任务、扫码确认、异常上报 |
| 复核应用服务 | 校验 SKU、数量、订单、包裹 |
| 打包应用服务 | 包装、称重、获取或绑定面单 |
| 盘点应用服务 | 生成盘点任务、提交差异 |

### 10.3 与库存服务关系

- 入库完成后通知库存服务增加库存。
- 出库完成后通知库存服务扣减库存。
- 盘点差异由库存服务最终生效。
- WMS 不能直接修改库存余额。

## 11. TMS 物流服务

### 11.1 职责

- 物流商账号和物流产品。
- 物流规则匹配。
- 发运单、包裹、面单。
- 轨迹同步。
- 物流账单和费用估算。

### 11.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 物流匹配应用服务 | 根据地址、重量、体积、商品属性匹配物流渠道 |
| 面单应用服务 | 获取、取消、重新生成面单 |
| 发运应用服务 | 创建发运单和包裹 |
| 轨迹应用服务 | 查询、订阅、导入物流轨迹 |
| 费用应用服务 | 运费估算、账单导入和差异核对 |

### 11.3 适配器

物流商适配器必须统一以下能力：

- 创建面单。
- 取消面单。
- 查询轨迹。
- 查询价格。
- 下载面单文件。

不同物流商字段差异不能泄漏到订单服务和仓储服务。

## 12. 采购服务

### 12.1 职责

- 供应商。
- 采购计划。
- 采购单。
- 到货通知。
- 质检和采购退货。

### 12.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 供应商应用服务 | 供应商档案、账期、评分和联系人 |
| 采购计划应用服务 | 基于销量、库存和在途生成采购建议 |
| 采购单应用服务 | 创建、审核、变更、关闭采购单 |
| 到货应用服务 | 生成到货通知并通知 WMS |
| 采购退货应用服务 | 管理不合格品退回供应商 |

## 13. 财务服务

### 13.1 职责

- 应收、应付。
- 平台结算。
- 采购成本、物流成本、仓储费用。
- 汇率。
- 利润核算。

### 13.2 应用服务

| 应用服务 | 说明 |
| --- | --- |
| 结算导入应用服务 | 导入平台结算报告并匹配订单 |
| 应收应用服务 | 管理订单应收、退款和调整 |
| 应付应用服务 | 管理采购、物流、仓储和平台费用 |
| 成本核算应用服务 | 计算 SKU、订单、店铺成本 |
| 利润应用服务 | 生成订单、SKU、店铺利润 |
| 汇率应用服务 | 维护汇率来源和生效时间 |

## 14. 报表服务

### 14.1 职责

- 经营报表。
- 库存报表。
- 仓储作业报表。
- 物流时效报表。
- 财务利润报表。

### 14.2 数据来源

- 订阅领域事件。
- 从业务库通过 CDC 同步。
- 定时从服务 API 拉取聚合数据。

优先推荐事件和 CDC，避免报表服务频繁调用业务服务造成耦合。

## 15. 文件服务

### 15.1 职责

- 商品图片。
- 面单文件。
- 发票。
- 导入导出文件。
- 报表文件。

### 15.2 关键规则

- 文件存对象存储，数据库只存元数据。
- 文件必须关联租户、业务类型、业务单号和访问权限。
- 临时文件需要过期清理。
- 敏感文件下载必须鉴权并记录审计。

## 16. 通知服务

### 16.1 职责

- 邮件、短信、站内信、Webhook。
- 告警通知。
- 模板管理。
- 发送记录和失败重试。

### 16.2 关键规则

- 通知发送必须异步。
- 模板变量必须校验。
- 失败可重试，超过阈值进入人工处理。
- Webhook 必须签名，避免被伪造。

## 17. 服务实现顺序

首期建议按以下顺序实现服务骨架和能力：

1. API 网关、认证权限服务、租户组织服务。
2. 商品服务、文件服务。
3. 渠道服务的订单导入能力。
4. 订单服务的导入、审核、查询能力。
5. 库存服务的余额、锁定、释放、扣减能力。
6. WMS 仓储服务的出库流程。
7. TMS 物流服务的物流规则和面单能力。
8. 通知服务和基础报表。
9. 采购服务和入库流程。
10. 财务服务和平台结算。
