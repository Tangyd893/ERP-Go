# 项目里程碑与全流程待办清单

## 1. 文档目标

本文档把 ERP-Go 从动工到竣工的实施路径拆成可执行里程碑，明确每个阶段的核心功能模块、工程任务、验收标准和 Todo 清单。本文档用于后续排期、拆 issue、分配人员、跟踪进度和验收交付。

## 2. 总体阶段

| 阶段 | 名称 | 目标 | 交付状态 |
| --- | --- | --- | --- |
| P0 | 项目启动与工程底座 | 完成技术栈、目录、规范、基础设施和服务模板 | 可开始写业务代码 |
| P1 | 账号租户与权限底座 | 完成登录、租户、组织、RBAC、审计 | 可进入业务系统 |
| P2 | 商品与渠道基础 | 完成 SKU、平台 SKU 映射、店铺授权、订单导入 | 平台订单可进入系统 |
| P3 | 订单与库存闭环 | 完成订单审核、库存余额、锁定、释放、扣减 | 订单可占用库存 |
| P4 | WMS 出库与 TMS 发货 | 完成拣货、复核、打包、面单、出库、回传 | 首期订单履约闭环完成 |
| P5 | 前端 MVP 与基础报表 | 完成管理后台、PDA、基础看板 | 业务人员可操作 |
| P6 | 采购与入库 | 完成供应商、采购、到货、质检、上架 | 采购补货闭环完成 |
| P7 | 财务结算与利润 | 完成应收应付、平台结算、成本、利润 | 经营核算闭环完成 |
| P8 | 测试加固与性能优化 | 完成自动化测试、压测、对账、异常补偿 | 可试运行 |
| P9 | 生产部署与竣工移交 | 完成部署、安全、备份、运维、培训、验收 | 可正式上线 |

- Vue 前端模板。
- Docker 中间件编排。
- 统一配置、日志、错误码、追踪、健康检查。
- 开发规范和提交规范。
